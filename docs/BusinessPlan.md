## 一、项目概述

### 1.1 一句话定位

**FinAgent Pro —— 让每一个金融从业者拥有AI驱动的"数字员工"团队。**

### 1.2 核心价值主张

FinAgent Pro 是基于大语言模型与多智能体协同技术的金融自主智能体平台。区别于传统金融信息终端的"被动查询"模式，FinAgent Pro 的6大专业智能体具备自主规划、决策与执行能力，能够像真正的金融分析师团队一样主动巡检市场、识别风险、生成策略并实时预警，推动金融服务从**被动响应**迈向**主动智能**。

我们解决的核心痛点：

- **信息过载，决策低效**：A股5000+标的、日均数万条新闻，人工分析力不从心
- **合规风险高企**：投资建议缺乏系统化合规审查，违规成本巨大
- **AI工具"不可信"**：黑箱推理无法审计，金融机构不敢用、不能用
- **专业人才稀缺**：持证分析师不足5万人，中小机构投研能力薄弱

### 1.3 产品愿景

成为中国金融行业最值得信赖的AI智能体基础设施，让合规、透明、高效的AI数字员工成为每一位金融从业者的标配。

## 三、产品方案

### 3.1 六大专业智能体

#### 1. 市场分析智能体（Market Analysis Agent）

- **能力**：技术指标分析（RSI/MACD/KDJ/布林带/ATR）、基本面评估、资金流向监控
- **工具**：实时行情获取、历史K线分析、技术指标计算、资金流向查询
- **输出**：多维度市场信号（强烈买入/买入/持有/卖出/强烈卖出）+ 置信度 + 关键发现

#### 2. 新闻舆情智能体（News Sentiment Agent）

- **能力**：中文新闻情感分析、社交媒体情绪提取、重大事件识别与影响评估
- **工具**：新闻搜索与摘要、情感评分、事件关联分析
- **输出**：舆情信号 + 事件影响评级 + 情绪趋势

#### 3. 风控合规智能体（Risk Compliance Agent）

- **能力**：多维风险评估（VaR/Beta/夏普比率）、合规规则引擎、集中度检查
- **工具**：风险指标计算、合规规则匹配、集中度分析
- **输出**：风险等级 + 合规检查结果 + 风险因子列表
- **合规规则**：单股集中度≤10%、行业集中度≤30%、创业板≤20%、ST股禁止买入

#### 4. 投资策略智能体（Strategy Agent）

- **能力**：资产配置建议、投资组合构建、策略回测
- **工具**：组合优化、相关性分析、历史回测
- **输出**：配置建议 + 目标价 + 止损价 + 持有周期

#### 5. 报告生成智能体（Report Agent）

- **能力**：研报/晨报/风控周报/事件快报自动生成
- **工具**：模板渲染、图表生成、数据可视化
- **输出**：结构化研报 + 可视化图表 + 关键结论

#### 6. 执行监控智能体（Execution Agent）

- **能力**：定时巡检、实时监控、阈值预警、主动推送
- **工具**：定时任务调度、实时数据流、预警规则引擎
- **输出**：预警通知 + 异常标记 + 行动建议

### 3.2 总调度智能体（Master Orchestrator）

作为"团队Leader"，Orchestrator负责：

1. **意图理解**：解析用户请求，分类为股票分析/市场概览/风险检查/通用对话
2. **任务分解**：将复杂请求拆解为子任务，确定执行依赖关系
3. **并行调度**：市场分析+新闻舆情并行执行，风控合规依赖前置结果串行执行
4. **多智能体协商**：收集各智能体信号，加权置信度协商生成最终建议
5. **LLM增强推理**：调用大模型综合各智能体分析，生成专业投资建议
6. **合规终审**：确保最终建议通过合规规则引擎检查

### 3.3 六大核心创新

| # | 创新点 | 技术实现 | 行业价值 |
|---|--------|----------|----------|
| 1 | **ReAct推理引擎** | 每个智能体具备"思考→行动→观察"循环，不是简单工具调用，而是自主规划推理链 | AI从"工具"变为"员工"，具备自主决策能力 |
| 2 | **三层持久记忆** | 工作记忆（当前上下文）+ 情节记忆（历史经验）+ 语义记忆（金融知识库） | 数字员工"记住"用户偏好与历史，越用越懂你 |
| 3 | **主动智能** | 定时巡检 + 事件驱动 + 阈值预警，从被动回答到主动工作 | 7×24小时不间断监控，不错过任何市场异动 |
| 4 | **合规内嵌** | 监管规则引擎 + 合规审计中间件，每步操作可审计可追溯 | 让AI建议"敢用、能用、合规" |
| 5 | **多智能体协商** | 6大专业智能体协同分析，加权置信度协商生成最终建议 | 模拟真实投研团队协作，多视角交叉验证 |
| 6 | **思维链可视化** | 实时展示AI推理过程（Thought→Action→Observation），让决策"看得见" | 解决AI黑箱问题，建立用户信任 |

### 3.4 技术架构

```
用户交互层:  Vue3 + TailwindCSS (暗色主题) + TradingView图表
    ↕  WebSocket / SSE 实时推送
API网关层:   FastAPI + 合规审计中间件 + 速率限制
    ↕
智能体层:    6大专业智能体 + Master Orchestrator
    ↕
推理框架层:  ReAct引擎 + Tool系统 + 三层Memory系统
    ↕
基础服务层:  LLM服务(DashScope通义千问/GLM-5.1/DeepSeek/SenseNova四模型降级) + DataAdapter(AKShare+yfinance+Finnhub+OpenBB统一适配) + PostgreSQL
```

**数据适配层**：FinAgent Pro 采用统一数据适配层(DataAdapter)设计，支持多数据源自动路由和降级。A股数据通过AKShare获取实时行情和历史K线，美股/海外数据通过yfinance和Finnhub获取，OpenBB提供基本面和财报数据。所有数据源具备三层降级机制：真实数据→缓存→Mock模拟数据，确保系统在任何网络环境下均可正常工作。

## 五、技术壁垒

### 5.1 ReAct推理引擎

- **核心算法**：基于"思考(Thought)→行动(Action)→观察(Observation)"循环的自主推理框架，每个智能体可独立规划推理链，动态调用工具，直至任务完成
- **差异化**：不同于简单的Prompt→Response模式，ReAct引擎让AI真正具备"规划-执行-反思"能力，可处理多步骤复杂金融分析任务
- **技术深度**：支持最大10步推理循环，上下文自动裁剪（6000 token窗口），结构化输出保障
- **四模型降级链**：DashScope(qwen-plus) → GLM-5.1 → DeepSeek-v4-pro → SenseNova，当主模型不可用时自动切换至下一模型，确保推理服务高可用

### 5.2 三层记忆系统

| 记忆类型 | 功能 | 技术实现 |
|----------|------|----------|
| **工作记忆** | 当前对话/任务的临时上下文 | 容量50条，LRU淘汰，2000 token上下文窗口 |
| **情节记忆** | 历史任务执行记录 | 100条episode，按任务类型检索相似经验 |
| **语义记忆** | 金融领域知识库 | 内置技术指标/风险指标/合规规则/行业分类知识图谱 |

**壁垒价值**：三层记忆让数字员工具备"经验积累"能力——分析过茅台的智能体，下次分析白酒板块时能调用历史经验，这是传统无状态AI无法实现的。

### 5.3 合规规则引擎

- **实时检查**：每次投资建议生成前自动执行合规审查（单股集中度、行业集中度、ST股禁止、创业板/科创板限制）
- **全链路审计**：合规审计中间件记录所有涉及投资建议的API调用，包含时间戳、用户IP、处理时长
- **可扩展性**：规则引擎支持动态配置，可快速适配新监管政策

### 5.4 技术护城河总结

| 维度 | 壁垒 | 竞对复制难度 |
|------|------|-------------|
| ReAct推理框架 | 自主规划+工具调用+反思循环 | 高（需深度Agent工程经验） |
| 三层记忆系统 | 工作记忆+情节记忆+语义记忆 | 高（需金融领域知识建模） |
| 合规规则引擎 | 监管规则+审计日志+可追溯 | 中高（需金融合规专业知识） |
| 多智能体协商 | 加权置信度+LLM增强推理 | 高（需协调工程+金融理解） |
| 思维链可视化 | 实时推理步骤展示 | 中（前端工程+推理框架配合） |
